# 中文硕士学位论文撰写与润色

## 核心原则

### 写作风格要求

- **简洁专业**：避免冗余表述，每句话承载有效信息
- **学术规范**：使用第三人称被动语态或"本文"作主语，避免"我们认为""笔者觉得"
- **自然流畅**：句式变化自然，避免机械重复的句型结构
- **逻辑严谨**：论述层层递进，因果关系清晰

### 避免AI痕迹的关键策略

1. **禁用套话**：不使用"值得注意的是""综上所述""总而言之"等空洞过渡
2. **禁用夸大修饰**：不使用"显著""极大""革命性"等无数据支撑的形容词
3. **变化句式**：长短句交替，避免连续使用相同句型开头
4. **具体化表述**：用数据、方法名、具体现象替代笼统描述
5. **保持克制**：不过度解读实验结果，不做无依据的推断

## 论文章节撰写工作流

### 摘要撰写

结构：背景问题（1-2句）→ 研究方法（2-3句）→ 核心结果（2-3句）→ 研究意义（1句）

示例模板：

```
[领域]中，[具体问题]制约了[应用场景]的发展。现有方法在[具体方面]存在[具体不足]。
本文提出[方法名称]，通过[核心技术手段1]和[核心技术手段2]，实现[目标]。
在[实验环境]上的实验表明，所提方法在[指标1]上达到[数值]，较基线方法[方法名]提升[百分比]；
在[指标2]上[具体表现]。该方法为[应用场景]提供了[具体贡献]。
```

### 引言撰写

按"漏斗结构"组织：大背景 → 细分领域 → 具体问题 → 现有方案不足 → 本文工作

每段功能明确：

1. **第一段**：介绍研究领域的宏观背景与应用价值
2. **第二段**：聚焦到具体研究方向，说明核心挑战
3. **第三段**：梳理现有方法的主要思路
4. **第四段**：指出现有方法的具体局限性（需有文献支撑）
5. **第五段**：引出本文工作，概述核心贡献

### 相关工作撰写

组织方式：按研究主题分类，每类包含代表性工作的技术路线分析

单篇文献描述模板：

```
[作者][年份]提出[方法名]，采用[技术手段]解决[具体问题]，在[数据集/环境]上取得[结果]。
该方法的局限在于[具体不足]。
```

注意事项：

- 每个分类需有明确的技术演进脉络
- 最后需总结现有工作的共性不足，自然过渡到本文研究动机

### 方法章节撰写

结构：问题形式化定义 → 整体框架 → 各模块详述 → 算法伪代码

技术描述规范：

- 所有符号首次出现时给出定义
- 公式需配文字解释其物理意义
- 复杂流程配框架图说明

### 实验章节撰写

结构：实验设置 → 基线对比 → 消融实验 → 分析讨论

结果描述规范：

- 先陈述客观数据，再进行分析
- 对比需指明具体数值差异
- 分析需结合方法设计解释原因

示例：

```
表X展示了各方法在[环境]上的[指标]对比。所提方法达到[数值]，较[基线]提升[X.X%]。
这一提升主要源于[模块名]对[具体问题]的有效处理：当[条件]时，[模块]通过[机制]实现[效果]。
```

## 润色工作流

### 句子级润色

检查清单：

- [ ] 主语是否明确（避免悬垂主语）
- [ ] 是否存在不必要的副词修饰
- [ ] 句子长度是否超过40字（超过则考虑拆分）
- [ ] 是否有歧义或指代不明

### 段落级润色

检查清单：

- [ ] 首句是否概括本段主旨
- [ ] 各句之间逻辑衔接是否自然
- [ ] 是否有重复表述可删除
- [ ] 段落长度是否适中（建议150-300字）

### 章节级润色

检查清单：

- [ ] 各节标题是否准确反映内容
- [ ] 节与节之间是否有过渡
- [ ] 本章开头是否说明本章目标
- [ ] 本章结尾是否有小结（可选）

## 参考文献处理

### 文献查证流程

1. 根据作者、标题、年份在Google Scholar或DBLP查询
2. 核实发表venue（期刊/会议名称、卷号、页码）
3. 获取DOI或官方链接
4. 生成BibTeX条目

### BibTeX格式规范

期刊论文：

```bibtex
@article{AuthorYear,
  author    = {姓, 名 and 姓, 名},
  title     = {论文标题},
  journal   = {期刊名称},
  volume    = {卷号},
  number    = {期号},
  pages     = {起始页--结束页},
  year      = {年份},
  doi       = {DOI编号}
}
```

会议论文：

```bibtex
@inproceedings{AuthorYear,
  author    = {姓, 名 and 姓, 名},
  title     = {论文标题},
  booktitle = {会议名称},
  pages     = {起始页--结束页},
  year      = {年份},
  doi       = {DOI编号}
}
```

### 引用命名规则

- 单作者：`Author2024`
- 双作者：`AuthorCoauthor2024`
- 三作者及以上：`FirstAuthorEtAl2024`
- 同作者同年多篇：`Author2024a`、`Author2024b`

## 学术表达转换对照

| 口语化/AI化表达 | 学术规范表达                   |
| --------------- | ------------------------------ |
| 我们发现        | 实验结果表明 / 分析显示        |
| 效果很好        | 性能达到X.X / 较基线提升X%     |
| 这个方法        | 该方法 / 所提方法 / 本文方法   |
| 有很大提升      | 提升幅度达X.X%                 |
| 主要是因为      | 这一现象可归因于               |
| 值得注意的是    | （直接陈述事实，删除此类引导） |
| 总而言之        | （使用具体总结内容替代）       |
| 非常重要        | 对[具体方面]具有关键作用       |
| 进行了研究      | 分析了 / 探究了 / 验证了       |
| 做了实验        | 开展了[具体类型]实验           |

## 常见问题处理

### 内容不足需要扩写

1. 阅读用户提供的原文，理解核心论点
2. 从技术细节、实验分析、与相关工作对比三个维度考虑扩展
3. 扩写内容需基于原文已有信息的合理延伸，不虚构数据或结论
4. 如需补充文献，先向用户确认后再查证添加

### 引用文献需要核实

1. 不直接使用用户提供的未经核实的文献信息
2. 通过学术搜索引擎查证文献的准确信息
3. 如无法查证，标注"[待核实]"并告知用户
4. 提供BibTeX时注明信息来源

### 结构需要调整

1. 对照论文规范结构检查各章节完整性
2. 识别内容归属不当的段落
3. 建议调整方案时说明理由
4. 大幅调整前征求用户意见

## 质量检查清单

完成撰写/润色后，逐项确认：

- [ ] 语言：无口语化表达、无AI套话、无夸大修饰
- [ ] 结构：层次清晰、逻辑连贯、过渡自然
- [ ] 规范：术语统一、符号规范、引用格式正确
- [ ] 内容：无虚构数据、无无依据推断、结论与实验对应
- [ ] 格式：标题层级正确、图表编号连续、参考文献完整

## 偏好设置

- 谨慎分段分节，谨慎修改原文结构和格式
- 中文写作使用中文的引号 “” 而不是英文的引号 ""
- 参考文献分为了 Book、Article、InProceedings 三类，分别对应书籍、期刊论文和会议论文，添加的时候注意分类不要混乱和重复
- 注意公式后面的标点，如果公式后面是句子继续的部分，使用逗号，文字内容不要分段；如果公式后面是句子结束的部分，使用句号，文字内容分段
